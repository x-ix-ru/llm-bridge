# План рефакторинга генерации OpenCode-конфигурации

---

## Цель

Вынести хардкод-константы лимитов контекстного окна (`defaultGenerationLimit = 3000`, buffer = 4000) в конфигурацию и перейти на формулу `context = buffer + input + output`, где output вычисляется как `context - buffer - input`.

---

## Список задач

| ID  | Название | Стэк | Статус |
|-----|----------|------|--------|
| 001 | Добавить поля конфигурации `opencode_context_buffer` и `opencode_context_input` | backend | new |
| 002 | Рефактор `handleOpenCodeConfig()` — использовать новую формулу и конфиг | backend | new |
| 003 | Обновить и дополнить тесты OpenCode Config | backend | new |
| 004 | Обновить документацию `configuration.md` | docs | new |

---

## Порядок выполнения и зависимости

```
001 ──► 002 ──► 003
                      │
004 ──────────────────┘
```

- **001** — нет зависимостей, запускается первым.
- **002** — зависит от 001 (нужны новые поля в конфиге).
- **003** — зависит от 001 и 002 (тестирует полный цикл: конфиг → формула → результат).
- **004** — независима, может быть выполнена параллельно или в конце.

### Критический путь

001 → 002 → 003

## Ожидаемый результат

После выполнения всех задач:
- Буфер контекста (default 4000) и аллокация input (default auto) задаются через `config.yaml`.
- Формула `context = buffer + input + output` строго выполняется.
- При значениях по умолчанию output = 3000 (backward compatible).
- Все существующие тесты проходят, добавлены новые тесты на формулу.
- Документация обновлена.
