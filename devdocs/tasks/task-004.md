# Task 004 — Тесты ENV override в config/config_test.go

## Описание

Добавить комплексные unit-тесты для логики ENV override: тестирование helpers (`envInt`, `envString`), переопределения в `Store.Load()`, игнорирование невалидных значений.

## Что решает

- Гарантирует корректность ENV-override логики.
- Покрытие всех 9 ENV-переменных, включая edge cases.
- Защита от регрессии при изменении логики загрузки.

## Покрытие требований

- Тесты `envInt()` и `envString()` — валидные и невалидные значения.
- Тесты `Store.Load()` с ENV override — ENV переопределяет YAML.
- Тесты приоритета: когда ENV не установлен, используется YAML.
- Тесты fallback: невалидный ENV → YAML-значение сохраняется.
